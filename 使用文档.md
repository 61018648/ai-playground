# 摘星AI 生图网站使用文档

本文档说明如何在本地运行当前项目，包括 Go 后端、PostgreSQL 数据库、Nuxt 前端和后台管理系统。

## 项目结构

```txt
E:\my-ai-image-app
├─ backend   # Go API 服务，连接 PostgreSQL
├─ frontend  # Nuxt 4 前端项目
└─ 使用文档.md
```

## 环境要求

- PostgreSQL，当前使用端口 `5432`
- Go，当前已验证 `go1.26.4`
- Node.js，当前已验证 `v24.18.0`
- Corepack，用于运行 `pnpm`

## 启动后端

进入后端目录：

```powershell
cd E:\my-ai-image-app\backend
```

执行数据库迁移：

```powershell
$env:DATABASE_URL="postgres://postgres:123456@127.0.0.1:5432/image-ai?sslmode=disable"
go run ./cmd/migrate
```

启动后端 API：

```powershell
$env:DATABASE_URL="postgres://postgres:123456@127.0.0.1:5432/image-ai?sslmode=disable"
$env:JWT_SECRET="local-dev-secret"
$env:HTTP_ADDR=":8080"
go run ./cmd/api
```

后端默认地址：

```txt
http://127.0.0.1:8080
```

健康检查：

```txt
http://127.0.0.1:8080/healthz
```

## 启动前端

进入前端目录：

```powershell
cd E:\my-ai-image-app\frontend
```

启动开发服务器：

```powershell
corepack pnpm dev --host 127.0.0.1 --port 3000
```

前端默认地址：

```txt
http://127.0.0.1:3000
```

## 注意事项

- 前端 API 地址写在 `frontend/nuxt.config.ts` 的 `runtimeConfig.public.apiBase` 中，当前为 `http://127.0.0.1:8080/api/v1`。
- 开发环境发送验证码时，后端会返回 `devCode`，方便本地测试。
- 当前生图结果是占位图片，后续接入真实生图模型时替换后端 `CompleteGenerationPlaceholder` 逻辑即可。
- 前端构建时可能出现 Google 字体元数据连接超时警告，目前不影响构建结果和本地运行。

## 推荐启动顺序

1. 启动 PostgreSQL。
2. 执行后端迁移。
3. 启动 Go 后端。
4. 启动 Nuxt 前端。
5. 打开 `http://127.0.0.1:3000/admin` 使用管理员账号登录后台。
